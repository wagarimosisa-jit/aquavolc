🌍Why AquaVolc?

The Problem with Volcanic Aquifers

Volcanic aquifers supply drinking water to over 200 million people globally, yet they remain among the most challenging hydrogeological systems to model due to:

  • Fracture-Controlled Flow: Transmissivity can vary by 2–3 orders of magnitude over short distances (e.g., <100 meters).
  • Extreme Spatial Heterogeneity: Interbedded layers of lava, tuff, ash, and clay create highly variable hydraulic properties.
  • Limited Data Availability: In developing countries, 80% of pumping test data is discarded because it lacks observation wells.

Why Existing Tools Fail

Current groundwater modeling tools (e.g., MODFLOW, INOWAS, HydroShare) do not address these challenges because they:

Limitation Impact
❌ No GPS enforcement Spatial errors in data, leading to incorrect models
❌ No single-well test support 80% of data is wasted (most common test type in developing countries)
❌ Require expert knowledge Inaccessible to engineers in developing countries
❌ Expensive/proprietary Unaffordable for governments and NGOs

✨Core Features

AquaVolc is the first open-source, web-based platform designed specifically for volcanic aquifers. Here’s how it solves the problem:

Feature Description Status
📍 GPS Enforcement Every data upload requires latitude, longitude, and elevation ✅ Implemented
🌍 Google Earth Integration Real-time verification of well locations on satellite imagery ✅ Implemented
📊 Single-Well Pumping Tests Estimates transmissivity (T) from specific capacity (SC) using:
log₁₀(T) = 0.98 × log₁₀(SC) – 0.06 (R² = 0.89, validated on Jimma dataset, Ethiopia) 		✅ Implemented
🔬 Multi-Well Tests Supports ≥2 observation wells, each with GPS coordinates ⏳ In Progress
📚 20+ Analytical Solutions Theis, Cooper-Jacob, Hantush-Jacob, Neuman, Warren-Root, Kazemi, etc. ⏳ In Progress
🤖 Automatic Solution Detection Hybrid rule-based + Random Forest ML (trained on 10,000 synthetic tests) ⬜ Planned
📝 NLP Lithology Parsing Converts free-text drilling logs into structured aquifer/aquitard layers ⬜ Planned
🗺️ Web-Based Kriging Temporal and spatial interpolation using GPS coordinates ⬜ Planned
📈 20+ Graphical Outputs Piezometric maps, type curves, water balance diagrams, 3D parameter cubes ⬜ Planned
📄 High-Resolution Exports 600 DPI PNG, SVG vector graphics, PDF reports ⬜ Planned

🤝How to Contribute

We welcome developers, hydrogeologists, and data scientists! No prior experience required—we’ll guide you.

💻 For Developers

No hydrogeology knowledge needed! We provide all the scientific details.

  1. Fork the repository: Click here to fork.
  2. Clone your fork: 
    git clone https://github.com/YOUR_USERNAME/aquavolc.git
cd aquavolc
    (Replace YOUR_USERNAME with your GitHub username.)
  3. Create a feature branch: 
    git checkout -b feature/your-feature-name
    Example branch names:
    • feature/theis-solution
    • feature/excel-parser
    • feature/google-earth-integration
  4. Set up the project: See our CONTRIBUTING.md guide.
  5. Pick a task: Browse our GitHub Issues. Beginner-friendly issues are labeled good first issue.
  6. Make your changes and test locally.
  7. Commit your changes: 
    git add .
git commit -m "Add your descriptive message here"
    Example: git commit -m "Implement Theis solution in FastAPI"
  8. Push to your fork: 
    git push origin feature/your-feature-name
  9. Open a Pull Request: Click here to create a PR.

📌 Good First Issues for Developers:

Task Description Estimated Time Difficulty
Implement Theis Solution Add the Theis (1935) analytical solution in Python 3–5 days ⭐⭐
